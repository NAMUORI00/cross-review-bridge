# Application Policy

Apply external feedback with the smallest local change that addresses the verified issue.

Before editing:

- quote or summarize the feedback item
- identify local evidence
- choose `apply`, `consider`, `reject`, or `needs user decision`
- state the intended change

Do not apply:

- broad rewrites without local evidence
- invented APIs, citations, experiments, files, or requirements
- changes that weaken security/privacy
- changes that overclaim research or product guarantees
- changes that require user tradeoff decisions

After editing:

- run relevant verification
- report what changed
- report rejected and deferred feedback
